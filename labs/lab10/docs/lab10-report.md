# 10.2 Указание к работе

  В большинстве дистрибутивов Linux в качестве текстового редактора по умолчанию устанавливается интерактивный экранный редактор vi (Visual display editor).
  Редактор vi имеет три режима работы:
– командный режим — предназначен для ввода команд редактирования и навигации по редактируемому файлу;
– режим вставки — предназначен для ввода содержания редактируемого файла;
– режим последней (или командной) строки — используется для записи изменений в файл и выхода из редактора.
  Для вызова редактора vi необходимо указать команду vi и имя редактируемого файла:
vi <имя_файла>
  При этом в случае отсутствия файла с указанным именем будет создан такой файл.
  Переход в командный режим осуществляется нажатием клавиши Esc . Для выхода из редактора vi необходимо перейти в режим последней строки: находясь в командном режиме, нажать Shift-; (по сути символ : — двоеточие), затем:
– набрать символы wq, если перед выходом из редактора требуется записать изменения в файл;
– набрать символ q (или q!), если требуется выйти из редактора без сохранения.
Замечание. Следует помнить, что vi различает прописные и строчные буквы при наборе
(восприятии) команд.

### 10.2.1.2. Команды позиционирования

- 0(ноль) - переход в начало строки;
- $ - переход в конец строки;
- G - перезод в конец файла;
- nG - переход на строку с номером n.

### 10.2.1.3. Команды перемещения по файлу

- Ctrl-d - перейти на пол-экрана вперёд;
- Ctrl-u - перейти на пол-экрана назад;
- Ctrl-f - перейти на страницу вперёд;
- Ctrl-b - перейти на страницу назад.

### 10.2.1.4. Команды перемещения по словам

- W или w - перейти на слово вперёд;
- nW или nw - перейти на n слов вперёд;
- b или B - перейти на слово назад;
- nb или nB - перейти на n слов назад.

## 10.2.2 Команды редактирования
### 10.2.2.1. Вставка текста

- a - вставить текст после курсора;
- A - вставить текст в конец строки;
- i - вставить текст перед курсором;
- ni - вставить текст n раз;
- I - вставить текст в начало строки.

### 10.2.2.2. Вставка строки

- o - вставить строку под курсором;
- O - вставить строку над курсором.

### 10.2.2.3. Удаление текста

- x - удалить один символ в буфер;
- d w - удалить одно слово в буфер;
- d $ - удалить в буфер текст от курсора до конца строки;
- d 0 - удалить в буфер текст от начала строки до позиции курсора;
- d d - удалить в буфер одну строку;
- n d d - удалить в буфер n строк.

### 10.2.2.4. Отмена и повтор произведённых изменений

- u - отменить последнее изменение;
- . - повторить последнее изменение.

### 10.2.2.5. Копирование текста в буфер

- Y - скопировать строку в буфер;
- nY - скопировать n строк в буфер;
- y w - скопировать слово в буфер;

### 10.2.2.6. Вставка текста из буфера

- p - вставить текст из буфера после курсора;
- P - вставить текст из буфера перед курсором.

### 10.2.2.7. Замена текста

- c w - заменить слово;
- n c w - заменить n слов;
- c $ - заменить текст от курсора до конца строки;
- r - заменить слово;
- R - заменить текст.

### 10.2.2.8. Поиск текста

- /текст - произвести поиск вперёд по тексту указанной строки символов текст;
- ?текст - произвести поиск назад по тексту указанной строки символов текст.

## 10.2.3 Команды редактирования в режиме командной строки
### 10.2.3.1. Копирование и перемещение текста

- :n,m d - удалить строки с n по m;
- :i,j m k - переместить строки с i по j, начиная со строки  k;
- :i,j t k - копировать строки с i по j в строку k;
- :i,j w имя-файла - записать строки с i по j в файл с именем имя-файла.

### 10.2.3.2. Записать в файл и выход из редактора

- : w - записать изменённый текст в файл, не выходя из vi;
- : w имя-файла - записать изменённый текст в новый файл с именем имя-файла;
- : w ! имя-файла - записать изменённый текст в файл с именем имя-файла;
- : w q - записать изменения в файл и выйти из vi;
- : q - выйти из редактора vi;
- : q ! - выйти из редактора без записки;
- : e ! - вернуться в командный режим, отменив все изменения, произведённые со времени последней записи.

## 10.2.4 Опции

  Опции редактора vi позволяют настроить рабочую среду. Для задания опций используется команда set (в режиме последней строки):
- : set all — вывести полный список опций;
- : set nu — вывести номера строк;
- : set list — вывести невидимые символы;
- : set ic — не учитывать при поиске, является ли символ прописным или строчным.
  Если вы хотите отказаться от использования опции, то в команде set перед именем
опции надо поставить no.

## 10.4 Ответ на контрольные вопросы

1. Дайте краткую характеристику режимам работы редактора vi
Редактор vi имеет три основных режима:

    Командный режим — режим по умолчанию. Предназначен для ввода команд редактирования, навигации по файлу и управления курсором.
    Режим вставки — предназначен для ввода и редактирования содержания файла. Все нажимаемые клавиши воспринимаются как текст.
    Режим последней строки — активируется нажатием : из командного режима. Используется для сохранения файла, выхода из редактора, поиска и выполнения расширенных команд.

2. Как выйти из редактора, не сохраняя произведённые изменения?

    Нажмите Esc (переход в командный режим)
    Введите :q!
    Нажмите Enter

3. Назовите и дайте краткую характеристику командам позиционирования

|-----------------------------------------------
| Команда    		Описание	
|-----------------------------------------------
| 0 (ноль)   	Переход в начало текущей строки
| $	     	    Переход в конец текущей строки
| G	     	    Переход в конец файла
| nG	        	    Переход на строку с номером n


4. Что для редактора vi является словом?
Понятие «слово» зависит от регистра команды:

    При использовании прописных W и B разделителями считаются только пробел, табуляция и возврат каретки.
    При использовании строчных w и b разделителями также считаются любые знаки пунктуации.

5. Каким образом из любого места редактируемого файла перейти в начало(конец) файла?

    В начало файла: 1G или gg
    В конец файла: G (без префикса)

6. Назовите и дайте краткую характеристику основным группам команд редактирования

    Вставка текста (i, I, a, A) — ввод символов до/после курсора или в начало/конец строки.
    Вставка строки (o, O) — создание новой пустой строки под/над курсором.
    Удаление текста (x, dw, dd, d$, ndd) — удаление символов, слов или строк с сохранением в буфер.
    Отмена и повтор (u, .) — отмена последнего действия или его повторение.
    Копирование в буфер (Y, nY, yw) — копирование строк или слов без удаления.
    Вставка из буфера (p, P) — вставка скопированного/удалённого текста после/перед курсором.
    Замена текста (cw, c$, r, R) — замена части строки или символа.
    Поиск текста (/текст, ?текст) — поиск вперёд или назад по файлу.

7. Необходимо заполнить строку символами $. Каковы ваши действия?

    Нажмите i (переход в режим вставки)
    Введите символ $ нужное количество раз
    Нажмите Esc (возврат в командный режим)
    При необходимости повторите действие командой . (точка)

Альтернатива: использовать режим замены R и удерживать клавишу $, либо команду c$ с последующим вводом символов.

8. Как отменить некорректное действие, связанное с процессом редактирования?
Нажмите u в командном режиме. Команда отменяет последнее изменение. Повторное нажатие u отменяет предыдущие изменения (поддерживается несколько уровней отмены в современных реализациях vi/vim).

10. Как определить, не перемещая курсора, позицию, в которой заканчивается строка?
В классическом vi визуального индикатора конца строки нет. Однако можно:

    Включить отображение невидимых символов командой :set list. В этом режиме в конце каждой строки будет отображаться символ $.
    Использовать $ для мгновенного перехода к концу строки (если допустимо перемещение курсора).
    Опираться на визуальный перенос строки в терминале.

11. Выполните анализ опций редактора vi (сколько их, как узнать их назначение и т.д.)

    Просмотр всех опций: :set all выводит полный список доступных опций и их текущие значения.
    Включение опции: :set <имя> (например, :set nu для нумерации строк, :set list для отображения спецсимволов, :set ic для игнорирования регистра при поиске).
    Отключение опции: перед именем ставится no (например, :set nonu, :set noic).
    Количество опций: зависит от конкретной реализации (vi, vim, neovim). В базовом vi их несколько десятков, в vim — более сотни. Назначение каждой опции можно узнать через :help <имя_опции>.

12. Как определить режим работы редактора vi?

    Командный режим: режим по умолчанию. Никаких индикаторов внизу экрана нет. Нажатие букв воспринимается как команды.
    Режим вставки: внизу экрана обычно появляется строка -- INSERT -- (или -- ВСТАВКА --). Все клавиши вводят текст.
    Режим последней строки: внизу экрана появляется приглашение : с мигающим курсором. Ввод воспринимается как ex-команда.
