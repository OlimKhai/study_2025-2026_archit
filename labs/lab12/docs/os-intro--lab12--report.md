	Архивирует (tar, zip или bzip2)

Код скрипта:

(
#!/bin/bash

# Создание директории backup
mkdir -p ~/backup

# Получение имени текущего скрипта
SCRIPT_NAME=$(basename "$0")

# Получение полного пути к скрипту
SCRIPT_PATH="$(readlink -f "$0")"

# Создание временной метки
TIMESTAMP=$(date +%Y%m%d_%H%M%S)

# Имя архива с временной меткой
BACKUP_NAME="${SCRIPT_NAME}.${TIMESTAMP}.tar.gz"

echo "============================================"
echo "Резервное копирование скрипта: $SCRIPT_NAME"
echo "============================================"
echo "Исходный файл: $SCRIPT_PATH"
echo "Архив: ~/backup/${BACKUP_NAME}"
echo ""

# Создание архива с помощью tar
tar -czf ~/backup/${BACKUP_NAME} -C "$(dirname "$SCRIPT_PATH")" "$(basename "$SCRIPT_PATH")"

# Проверка успешности создания
if [ $? -eq 0 ]; then
    echo "✓ Резервная копия успешно создана!"
    echo "✓ Размер архива: $(du -h ~/backup/${BACKUP_NAME} | cut -f1)"
else
    echo "✗ Ошибка при создании резервной копии!"
    exit 1
fi


)

(
#!/bin/bash

# Если аргумент не передан, используем текущую директорию
if [ $# -eq 0 ]; then
    TARGET_DIR="."
else
    TARGET_DIR="$1"
fi

# Проверка существования директории
if [ ! -d "$TARGET_DIR" ]; then
    echo "Ошибка: '$TARGET_DIR' не является директорией!"
    exit 1
fi

echo "Содержимое директории: $TARGET_DIR"
echo "------------------------------------------------------------"
printf "%-30s %-5s %-10s\n" "Имя файла" "Тип" "Права (R/W/X)"
echo "------------------------------------------------------------"

# Перебор всех файлов в директории
for file in "$TARGET_DIR"/*; do
    [ -e "$file" ] || continue
    
    filename=$(basename "$file")
    
    # Определение типа файла
    type_str="file"
    [ -d "$file" ] && type_str="dir"
    [ -L "$file" ] && type_str="link"
    
    # Проверка прав доступа
    if [ -r "$file" ]; then r="r"; else r="-"; fi
    if [ -w "$file" ]; then w="w"; else w="-"; fi
    if [ -x "$file" ]; then x="x"; else x="-"; fi
    
    perms="${r}${w}${x}"
    
    printf "%-30s %-5s %-10s\n" "$filename" "$type_str" "$perms"
done



)

#!/bin/bash

# Проверка количества аргументов
if [ $# -lt 2 ]; then
    echo "Ошибка: Недостаточно аргументов."
    echo "Использование: $0 <расширение> <путь_к_директории>"
    exit 1
fi

EXTENSION="$1"
TARGET_DIR="$2"

# Проверка существования директории
if [ ! -d "$TARGET_DIR" ]; then
    echo "Ошибка: Директория '$TARGET_DIR' не найдена."
    exit 1
fi

echo "Поиск файлов с расширением '$EXTENSION' в директории: $TARGET_DIR"
echo "------------------------------------------------------------"

COUNT=0

# Перебор файлов с указанным расширением
for file in "$TARGET_DIR"/*"$EXTENSION"; do
    if [ -f "$file" ]; then
        COUNT=$((COUNT + 1))
    fi
done

echo "------------------------------------------------------------"
echo "Всего найдено файлов: $COUNT"


)

## Контрольные вопросы (#контрольные-вопросы)

1. Объясните понятие командной оболочки. Приведите примеры командных оболочек. Чем они отличаются?

Командная оболочка (интерпретатор команд, shell) — это программа, позволяющая пользователю взаимодействовать с операционной системой, запуская команды и интерпретируя скрипты.

Примеры:

'
    sh (Bourne shell): Стандартная оболочка UNIX с базовым набором функций.
    csh (C-shell): Надстройка над sh, использующая C-подобный синтаксис и поддерживающая историю команд.
    ksh (Korn shell): Напоминает C-оболочку, но совместима с синтаксисом sh.
    BASH (Bourne Again Shell): Разработка FSF, совмещает свойства C-оболочки и Korn shell, является стандартом для Linux.
    Отличия: Они различаются синтаксисом команд, способами управления историей и возможностями программирования (хотя большинство базовых команд совпадает).
'

2. Что такое POSIX?

POSIX (Portable Operating System Interface for Computer Environments) — это набор стандартов, разработанных комитетом IEEE для описания интерфейсов взаимодействия ОС и приложений. Цель — обеспечить совместимость различных UNIX/Linux-подобных систем и переносимость прикладных программ на уровне исходного кода. POSIX-совместимые оболочки разработаны на базе Korn shell (ksh).


3. Как определяются переменные и массивы в языке программирования bash?

    Переменные: Определяются присваиванием значения строки символов без пробелов вокруг знака равенства: имя=значение (например, mark=/usr/andy/bin). Для доступа к значению используется знак доллара: $имя или ${имя}.
    Массивы: Создаются с помощью команды set -A имя_массива val1 val2 .... Индексация начинается с нуля. Добавить элемент можно так: имя_массива[49]=Alaska.

4. Каково назначение операторов let и read?

'
    let: Используется для арифметических вычислений. Он воспринимает следующие аргументы как выражение, подлежащее вычислению (например, let sum=x+7). Поддерживает сложение, вычитание, побитовые операции и т.д.
    read: Позволяет читать значения переменных со стандартного ввода (с клавиатуры). Например: read mon day считает два слова и запишет их в переменные mon и day.
'

5. Какие арифметические операции можно применять в языке программирования bash?

В bash поддерживаются стандартные математические и логические операции (см. Таблицу 10.1 в методичке):

'
    Арифметические: + (сложение), - (вычитание), * (умножение), / (целочисленное деление), % (остаток от деления).
    Логические и побитовые: && (AND), || (OR), ! (NOT), &, |, ^.
    Сравнения: <, >, <=, >=, ==, !=.
    Сдвиги: << (влево), >> (вправо).
'

6. Что означает операция (())?

Двойные круглые скобки используются для записи сложных арифметических выражений и условий без необходимости писать ключевое слово let. Внутри скобок ноль воспринимается как «ложь», а любое другое значение — как «истина». Это удобно для использования в условиях циклов, например: while (( x > 0 )).


7. Какие стандартные имена переменных Вам известны?

    PATH: Список каталогов для поиска команд.
    HOME: Путь к домашнему каталогу пользователя.
    IFS: Разделители в командной строке (пробел, табуляция, перевод строки).
    MAIL: Путь к файлу входящей почты.
    TERM: Тип используемого терминала.
    LOGNAME: Имя пользователя.
    PS1, PS2: Строки промптеров (приглашений командной строки).

8. Что такое метасимволы?

Метасимволы — это символы, имеющие для командного процессора специальный смысл (управляющие символы).

Примеры:

'
    * — соответствует любой строке (включая пустую).
    ? — соответствует любому одиночному символу.
    [c1-c2] — соответствует любому символу из диапазона.
    <, >, | — используются для перенаправления потоков ввода-вывода и конвейеров.
'

9. Как экранировать метасимволы?

Экранирование — это снятие специального смысла с метасимвола:

    Использованием обратного слэша \ перед символом (например, echo \* выведет просто звездочку).
    Заключением группы метасимволов в одинарные кавычки '...' (экранирует всё внутри).
    Заключением в двойные кавычки "..." (экранирует большинство метасимволов, но сохраняет спецсмысл $, \, " и `).

10. Как создавать и запускать командные файлы?

    Создать текстовый файл с набором команд.
    Запускать файл командой bash имя_файла [аргументы].
    Чтобы запускать файл просто по имени, нужно изменить права доступа, добавив бит исполнения: chmod +x имя_файла.

11. Как определяются функции в языке программирования bash?

Функции определяются с помощью ключевого слова function или круглых скобок:

function fname {
	команды
}
# или
fname() {
	команды
}

Для удаления функции используется команда unset -f fname.

12. Каким образом можно выяснить, является файл каталогом или обычным файлом?

С помощью команды test или квадратных скобок [ ... ]:

    test -f file (или [ -f file ]) — истина, если файл существует (обычный файл).
    test -d file (или [ -d file ]) — истина, если файл является каталогом.
    Также можно проверить права: test -r file (чтение), test -w file (запись), test -e file (исполнение).


13. Каково назначение команд set, typeset и unset?

    set: Выводит список всех переменных и функций; используется для создания массивов (set -A).
    typeset: Позволяет задавать атрибуты переменных. Например, typeset -i объявляет переменную целочисленной, typeset -f перечисляет функции, typeset -ft включает трассировку.
    unset: Изъятие переменной или функции из памяти (например, unset -f для удаления функции).


14. Как передаются параметры в командные файлы?


Параметры передаются при вызове скрипта через пробел (например, ./script arg1 arg2). Внутри скрипта они доступны как позиционные переменные: $0 (имя файла), $1 (первый аргумент), $2 (второй) и т.д. Для доступа к аргументам с номером больше 9 используются фигурные скобки: ${10}, ${11}. Специальная переменная $# хранит общее количество переданных параметров.


15. Назовите специальные переменные языка bash и их назначение.

'
    $# — количество аргументов.
    $* и $@ — все аргументы как единое целое (разница в обработке пробелов).
    $? — код завершения последней команды (0 — успех).
    $$ — уникальный идентификатор процесса (PID) текущего shell.
    $! — PID последнего процесса, запущенного в фоновом режиме.
    $- — текущие флаги командного процессора.
    ${#name} — длина строки в переменной name.
'
